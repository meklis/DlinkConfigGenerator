В этом каталоге сохраняются шаблоны конфигов.

==== Параметры загрузки шаблона ====
В каждом каталоге есть файл global.conf - он грузится в первую очередь, после него шаблон модели.
В случае обнаружения в модели блока с таким же именем в global, блок команд с файла global будет проигнорирован.


==== Конфигуратор генератора, описание обозначений ====
[SERVICE_NET] - Что бы передать массив вместо строки.  В этом случае команда выполнится несколько раз с подстановкой каждого элемента массива
# - Имя логического блока
@ - Команда, которая не требует проверки на успешность
; - Коментарий

==== Вланы ====
VLAN_INET - ID влана с именем INTERNET
VLAN_SWITCH - ID влана с именем switchesXXXX

==== Список портов ====
*_PORTS - список портов, переданный ввиде строки. Например 1-5, или 1,2,3,4,5 или 1-4,5
MAGISTRAL_PORTS - магистральные порты. По дефолту - аплинк-порт и FIBER порты(указаны в DeviceParams.json)
ABON_PORTS - абонентские порты
FIBER_PORTS - оптические порты
ALL_PORTS - все порты (количество портов указано в DeviceParams.json)

==== IP адреса или подсети ====
SERVICE_NET - сервисные подсети 109.86.2.0,80.73.9.0,80.73.11.0
SNTP_PRIMARY - основной sntp-сервер
SNTP_SECONDARY - резервный sntp-сервер
RADIUS_SERVER - IP радиус-сервера для аутификации IGMP
TRUSTED_NETWORKS - подсети в трасты, вида CIDR -  10.50.124.0/24
DHCP_SERVER_IP - IP dhcp relay сервера
